maze and the mouse

repository

https://github.com/LucasCerqueiraGalvao/trabalho_labirinto

Overview

This project showcases a C program designed to solve mazes using a stack-based depth-first search (DFS) algorithm. The program includes functions to initialize and manipulate a stack, load and validate a maze from a file, explore the maze to find an exit (represented as cheese 'S'), and display the maze in the console. Here's a breakdown of the key components.

Stack Operations

  1. initStack

    Initializes the stack.

  2. isStackEmpty

    Checks if the stack is empty. Returns 1 if empty, 0 otherwise.

  3. push

    Inserts an element at the top of the stack. Does nothing if the stack is full.

  4. pop

    Removes and returns the top element of the stack. Returns an invalid position (-1, -1) if the stack is empty.

Maze Loading and Validation

  1. loadMaze

    Loads a maze from a file. The file format includes the number of rows and columns, followed by the maze layout (0 for open path, X for wall, S for cheese).

  2. validateMaze

    Ensures the maze contains valid characters (0, X, S) and at least one exit (S).

Maze Exploration

  1. exploreMaze

    Uses a stack to perform a depth-first search from a starting position. Marks the path and finds the exit (S).

  2. isValidMove

    Checks if a move to a specified position in the maze is valid (either an open path 0 or the cheese S).

  3. findExit

    Initiates maze exploration from a given start position to find the exit.

Display Function

  1. Initialize Allegro

    Set up the Allegro library, including initializing display, event queue, and necessary add-ons.

  2. Load Resources

    Load images, fonts, and other resources required for rendering the maze and UI components.

  3. Render Loop

    Implement the main rendering loop to draw the maze, player position, and other elements, refreshing the display at a fixed frame rate.

  4. Event Handling

    Process user inputs and events such as keyboard presses and window close requests within the rendering loop.

  5. Clean Up

    Properly release resources and shut down Allegro components when the program exits.

Cesar Cipher

repository

https://github.com/LucasCerqueiraGalvao/Cifra-de-Cesar

Project Overview

This project focuses on encrypting and decrypting messages using a simple Caesar Cipher-like algorithm implemented in Python. The encryption process involves generating a random key for each character in the input message, while the decryption process requires the correct key to retrieve the original message. This project demonstrates skills in string manipulation, random number generation, and basic cryptography principles.

Encryption Process

  1. Input Message

    The user inputs the message to be encrypted.

  2. Character List

    The input message is converted into a list of characters.

  3. Key Generation

    A random key is generated for each character within a specified range (0-100).

  4. Encryption

    Each character's ASCII value is incremented by its corresponding key value, resulting in an encrypted message.

  5. Output Encrypted Message

    The encrypted message and the key used are printed.

Decryption Process

  1. Input Encrypted Message

    The user inputs the encrypted message as a list of integers.

  2. Input Key

    The user inputs the key used during encryption as a list of integers.

  3. Decryption

    Each encrypted character's value is decremented by its corresponding key value to retrieve the original ASCII value.

  4. Output Decrypted Message

    The decrypted message is printed, restoring the original input message.

Summary

This project involved developing a simple encryption and decryption algorithm using Python. The encryption process utilized random key generation and ASCII value manipulation to secure the message, while the decryption process required the correct key to retrieve the original message. This project showcases skills in Python programming, string manipulation, random number generation, and basic cryptography.
